The Service Bus view lets you browse namespaces, queues, and topics. You can
peek messages (read-only) and then optionally lock and manipulate dead-letter
messages — requeue them to the active queue, move them to a different target,
or complete them outright.

**Common tasks this replaces:**
- `az servicebus` commands chained together to peek and parse messages
- Opening Azure Portal's Service Bus Explorer to check dead-letter queues
- Writing scripts to requeue or clean up DLQ messages
- Manually counting message depths across multiple queues

## Pane layout

The view uses miller-columns that grow as you drill in:

**Namespaces** → **Entities** → **Subscriptions** (topics only) → **Active/DLQ** → **Messages** → **Preview**

- **Entities** are queues and topics for the selected namespace.
- **Subscriptions** only appears when you've selected a topic.
- **Active/DLQ** is a two-item picker where you choose which queue type to browse.
- **Messages** is empty by default — peek is a deliberate action, not automatic.
- **Preview** opens when you press `Enter` on a message.

Not all panes are visible at once — the view shows a parent, focused, and child
pane and shifts as you navigate.

## Browsing entities

Entities are queues and topics within a namespace. Press `Enter` on a namespace
to load its entities.

For **queues**, pressing `Enter` takes you to the Active/DLQ picker.

For **topics**, pressing `Enter` shows the topic's subscriptions. Select a
subscription to reach the Active/DLQ picker.

### Sorting and filtering entities

Press `a` to open the action menu and select **Sort entities** to sort by:

- Default order
- Name (ascending / descending)
- Active messages (ascending / descending)
- Dead letters (ascending / descending)

The action menu also offers **Filter: only with dead letters** to hide entities
with zero DLQ messages, and **Show all entities** to clear the filter. Press `s`
as a shortcut to toggle the DLQ filter.

## Peeking messages

After selecting Active or DLQ, the messages pane is empty. Peek is a deliberate
action — no network round-trip happens until you ask for it.

Open the action menu (`a`) from the messages pane and choose one of:

- **Peek messages** — fetch the first batch
- **Peek more messages** — append another batch, continuing from where the
  previous peek left off
- **Peek messages (fresh)** — discard the current batch and peek from the start
- **Clear messages** — drop the local snapshot without peeking

<Aside type="note">
Peeking is read-only. It does not lock or consume messages. You can peek the
same DLQ repeatedly without affecting it.
</Aside>

Press `Enter` on a peeked message to open the preview pane with the full
message body. You can select text in the preview with the mouse to copy it.

| Key | Action |
|---|---|
| `Enter` | View message detail |
| `h` / `Esc` / `Backspace` | Close message preview |
| `Space` | Mark / unmark message |
| `v` / `V` | Enter visual-line mode to select a range |
| `o` | Swap anchor while in visual mode |
| `Esc` | Commit visual selection as marks |

## Dead-letter queue operations

<Aside type="caution">
DLQ modifications happen in two modes: **peek** (read-only, default) and
**receive-with-lock** (actions that remove or move messages). The
lock-based flow uses Azure Service Bus's native peek-lock semantics so
operations are transactional. Closing the messages pane or navigating away
abandons any held locks automatically.
</Aside>

### Receive with lock

To actually change DLQ state, open the action menu and choose **Receive DLQ
messages (with lock)**. This pulls up to 50 messages from the DLQ and holds
locks on them. The pane title changes to `DLQ Locked (N)` to indicate the mode.

Once locked, the action menu offers:

- **Requeue N message(s) (send + complete)** — send each locked message to the
  active queue, then complete it from the DLQ
- **Move N message(s) to...** — send each locked message to a different target
  queue or subscription (picker opens), then complete from DLQ
- **Complete N message(s) (remove from DLQ)** — complete without re-sending
- **Abandon all (release locks)** — release all locks, messages stay in DLQ

`N` is the number of marked messages, or `1` (the cursor) if nothing is marked.
Use `Space` or visual-line mode to mark messages before opening the action menu.

### Bulk operations from the queue type pane

For larger jobs, skip the per-message lock flow and use the action menu on the
**queue type pane** (Active/DLQ picker):

- **Requeue all DLQ messages (N)** — drains the entire DLQ to the active queue.
  Receives in batches of 50, sends each, completes each, repeats until empty.
- **Move all messages to...** — same but sends to a different target you pick.

These are safer than looping locally because they lock each batch before
processing, and recover if interrupted (unprocessed messages retain their
original delivery count).

### Operation progress

Long-running bulk operations stream progress through the **stream manager**
(`F`), which shows status and lets you cancel mid-flight. When a stream is
active it appears in the notification bar.

## Selecting messages

Mark individual messages with `Space`, or use visual-line mode (`v`) to select
a range. Marked messages can be operated on in bulk through the action menu.

Visual-line mode behaves like vim:

1. Press `v` to anchor at the current message.
2. Move the cursor to extend the selection; press `o` to extend from the other end.
3. Press `v` again or `Esc` to commit the range to marks.

## Inspect panel

Press `K` to toggle the inspect strip, which shows details for the selected
namespace, entity, subscription, or message — including message counts,
metadata, and properties.

## Jump list

Service Bus tabs participate in the cross-tab jump list. Pressing `Ctrl+O`
jumps back to your previous navigable position (including a different tab);
`Ctrl+I` goes forward. See [Navigation](/lazyaz/navigation/#jump-list) for
details.

## Actions

Press `a` to open the action menu. Available actions depend on which pane is focused.

### Entities pane

| Action | Description |
|---|---|
| Sort entities | Sort by name, active count, or dead-letter count |
| Show all entities / Filter: only with dead letters | Toggle the DLQ entity filter |

### Queue type pane

| Action | Description |
|---|---|
| Requeue all DLQ messages (N) | Requeue every dead-letter message back to the active queue |
| Move all messages to... (N) | Move all active or DLQ messages to a different queue or subscription |

### Messages pane — peek mode

| Action | Description |
|---|---|
| Peek messages | Fetch a batch of messages (read-only) |
| Peek more messages | Append another batch |
| Peek messages (fresh) | Discard current batch and re-peek |
| Clear messages | Drop the local snapshot |
| Receive DLQ messages (with lock) | Lock messages for transactional operations (DLQ only) |
| Toggle mark | Mark or unmark the current message |
| Toggle visual line selection | Enter or exit visual-line mode |

### Messages pane — locked mode

| Action | Description |
|---|---|
| Requeue N message(s) | Send marked messages to the active queue and complete from DLQ |
| Move N message(s) to... | Send marked messages to a different target and complete from DLQ |
| Complete N message(s) | Complete (remove) marked messages from the DLQ |
| Abandon all (release locks) | Release all held locks without modifying messages |
| Toggle mark | Mark or unmark the current message |
| Toggle visual line selection | Enter or exit visual-line mode |

### Any pane

| Action | Description |
|---|---|
| Refresh | Reload the current scope |
| Toggle details panel | Show or hide the inspect strip |
| Change subscription | Open the subscription picker |
