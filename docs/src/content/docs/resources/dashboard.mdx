The Dashboard is a tiled overview that surfaces the state of your Azure
resources without drilling into individual explorer tabs. It's useful as a
landing page when you open a subscription â€” check DLQ depths, spot unusual
queue counts, and jump straight into whichever resource needs attention.

## Opening the dashboard

The dashboard is a tab kind, same as Blob / Service Bus / Key Vault. Add it to
your startup config:

```json
{
  "tabs": [
    { "kind": "dashboard", "subscription": "your-subscription-id" }
  ]
}
```

Or open one at runtime with the command palette (`Ctrl+P`) â†’ **New Dashboard
Tab**.

## Widgets

The dashboard shows four widgets tiled in a grid:

| Widget | Shows |
|---|---|
| **Namespace counts** | Message counts per Service Bus namespace â€” active and dead-letter totals across all entities |
| **DLQ alerts** | Entities with dead-letter messages, sorted by count. Each row is a direct navigation target |
| **Most used Blob** | Your frequently accessed blob containers â€” based on recent navigation history |
| **Most used Service Bus** | Your frequently accessed queues and subscriptions â€” same heuristic |

All widgets update automatically at a regular interval. They share a single
cache broker so the data is also available instantly to the matching
explorer tab.

Unlike the resource tabs, which use a linear pane focus cycle, the dashboard
uses **spatial navigation** â€” each widget is a tile and you move between them
like moving around a grid.

## Drilling into a resource

Press `Enter` on a row to open the matching explorer with that resource
already selected. For example, pressing `Enter` on a DLQ alert row opens a
Service Bus tab navigated directly to that entity's dead-letter pane.

This works through the same pending-navigation path the jump list uses â€” if
the target tab's cache is warm the switch is instant; otherwise it fetches
just what's needed to land you on the right pane.

## Widget actions

Some widgets have context-aware actions â€” open the action menu with `a` while
a widget is focused. For example, the **DLQ alerts** widget offers sort options
(by namespace, by entity, by count).

The action menu shows only the actions relevant to the focused widget. Press
`a`, navigate with `j`/`k`, `Enter` to confirm.

## Refresh

<Aside type="note">
The dashboard auto-refreshes its widgets on a timer, so you usually don't need
to refresh manually. Press `r` to force an immediate refresh of the focused
widget.
</Aside>

## Jump list

Dashboard positions aren't individually recorded in the jump list (widgets
don't have a deep navigational structure), but **opening** a dashboard tab
and **drilling out** into a resource tab are both tracked. You can `Ctrl+O`
back to the dashboard after drilling into an entity from a DLQ alert.

See [Navigation](/lazyaz/navigation/#jump-list) for details.
