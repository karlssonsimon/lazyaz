## Pane layout

lazyaz uses a **miller-columns** layout — parent, focused, and child panes
arranged left to right. As you drill into resources, the focused pane shifts
right and the previous pane becomes the parent column on the left.

|---|---|
| `Tab` / `Shift+Tab` | Cycle pane focus |

## Action menu

Press `a` to open a context-aware action menu for the current pane. The menu
shows only actions relevant to your current selection — download, yank, sort,
requeue, and more depending on the resource type. Type to filter, `Enter` to
execute.

## Command palette

Press `Ctrl+P` to open the command palette. It lists all available commands —
open tabs, switch themes, view notifications, and more. Type to filter,
`Enter` to execute.

## Jump list

lazyaz keeps a vim-style jump list that tracks where you've been across tabs.
Each time you drill into a namespace, container, or subscription, the position
is recorded. Switching tabs and navigating into resources is also recorded.

| Key | Action |
|---|---|
| `Ctrl+O` | Jump back to the previous position |
| `Ctrl+I` / `Ctrl+N` / `Ctrl+]` | Jump forward |

Jumps cross tab boundaries — pressing `Ctrl+O` can return you to a different
tab you had open. The list holds up to 100 entries and is deduplicated so
rapid back-and-forth doesn't bloat it.

Some terminal + multiplexer combinations can't distinguish `Ctrl+I` from `Tab`,
which is why `Ctrl+N` and `Ctrl+]` are also bound to jump forward.
